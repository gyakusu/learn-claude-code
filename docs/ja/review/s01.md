## レビュー結果

### 内容は本当か？

**はい、技術的に正確です。** 実装コード (`agents/s01_agent_loop.py`) とドキュメントの説明を照合した結果：

- **APIの使い方** — `client.messages.create()` に `tools` を渡し、`stop_reason` で分岐するパターンは Anthropic Messages API の正しい使い方
- **メッセージ構造** — `tool_result` を `role: "user"` で返す仕様も正確
- **ループ構造** — 実装コードとドキュメントのコードスニペットはほぼ一致している
- **「30行未満」** — `agent_loop` 関数本体は実装でも約20行で、誇張ではない

**1点だけ差異がある：** ドキュメントの「まとめた関数」では `agent_loop(query)` が引数として `query` (文字列) を受け取り内部で `messages` を作成しているが、実装では `agent_loop(messages)` が外部から `messages` リストを受け取っている。これはマルチターン対話を実現するための実用的な変更であり、コアのループパターンには影響しない。

### 最も大事な学習ポイント

1. **エージェント = ループ + ツール、それだけ**
   - `while True` → LLM呼び出し → `stop_reason` 確認 → ツール実行 → 結果を戻す。この骨格がすべてのAIコーディングエージェントの基盤。Claude Code自体もこの構造の上に構築されている。

2. **終了条件は `stop_reason != "tool_use"` の一点のみ**
   - モデルが「もうツールを呼ぶ必要がない」と判断した時点で止まる。つまり**モデル自身が完了判断をする**。ハーネス側は何も判断しない。

3. **ツール結果は `role: "user"` で返す**
   - これは直感に反するが、Anthropic APIの仕様。ツール実行結果を「ユーザーメッセージ」として追加することで会話が継続する。

4. **このループは以降のすべてのセッションの土台であり、変わらない**
   - ドキュメントが明示している通り、s02以降で追加されるのはこのループの「上」に積む層（ツール追加、権限制御、コンテキスト管理など）であって、ループ自体のパターンは不変。
